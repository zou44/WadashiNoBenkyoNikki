## 为什么要规范化?
关系数据库设计理论的`核心`是数据间的`函数依赖`，`衡量的标准`是`关系规范化程度`以及`无损连接`和`保持函数依赖性`。
(大白话: 关系数据库设计核心是函数依赖，若要判断关系的函数依赖设计的好坏则有一套衡量标准，就是用`无损连接`、`保存函数依赖`和`关系的规范化程度`来判定。)

## 什么是规范化？
通过模式分解，将一个低等级范式的关系模式转换成若干个高一级范式的关系模式，这个过程称为规范化。
